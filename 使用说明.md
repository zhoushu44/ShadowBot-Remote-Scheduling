# Webhook BAT 执行器使用说明

## 项目简介
这是一个基于 PyQt5 的 Webhook 应用程序，可以将 HTTP 请求转换为执行本地 BAT 脚本。通过 FRP 内网穿透技术，可以让外部网络访问你本地的服务。

## 快速开始

### 1. 环境准备
确保你的电脑已安装 Python 3.7+，如果没有安装，请先下载安装：
- 访问 [Python官网](https://www.python.org/downloads/) 下载最新版本
- 安装时勾选 "Add Python to PATH" 选项

### 2. 安装依赖
打开命令提示符（按 Win+R，输入 cmd，按回车），进入项目文件夹：
```bash
cd C:\Users\zhou\Desktop\测试22
```

安装必要的 Python 包：
```bash
pip install PyQt5 requests flask
```

如果下载太慢，可以使用国内镜像：
```bash
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple PyQt5 requests flask
```

### 3. 准备 BAT 脚本
1. 在 `新建文件夹/632882470291922946/` 目录下创建你的 BAT 脚本
2. 脚本文件名应该与 webhook 请求中的 key 值对应
3. 例如：当请求 `{"key": "22"}` 时，会执行 `22.bat` 文件

### 4. 启动应用程序
双击运行 `11.py` 文件，或者使用命令：
```bash
python 11.py
```

程序启动后会出现主界面，包含以下功能：
- **BAT 目录设置**：设置 BAT 脚本所在的文件夹
- **FRP 设置**：配置内网穿透参数
- **测试连接**：测试本地和外部连接状态
- **生成 curl**：生成测试用的 curl 命令

### 5. 配置 FRP 内网穿透（可选）
如果你需要让外部网络访问你的服务：

1. 点击主界面的 "FRP 设置" 按钮
2. 在 FRP 设置窗口中配置：
   - **服务器地址**：`117.72.35.239`
   - **服务器端口**：`7000`
   - **Token**：`60d8a83c544e6168db`
   - **远程端口**：`6000`
3. 点击 "保存配置"
4. 在主界面点击 "启动 FRP"

程序启动后会在系统托盘显示图标，支持：

### Webhook 接口说明

#### 请求格式
```http
POST http://127.0.0.1:8888/webhook
Content-Type: application/json

{
    "key": "脚本文件名"
}
```

#### 响应格式
```json
{
    "code": 0,
    "msg": "脚本执行结果"
}
```

#### 示例
本地测试：
```bash
curl -X POST http://127.0.0.1:8888/webhook -H "Content-Type: application/json" -d '{"key":"22"}'
```

外部测试（需要 FRP 正常运行）：
```bash
curl -X POST http://117.72.35.239:6000/webhook -H "Content-Type: application/json" -d '{"key":"22"}'
```

### BAT 脚本编写规范

1. **文件命名**：使用英文或数字命名，如 `test.bat`、`22.bat`
2. **编码格式**：保存为 UTF-8 或 ANSI 编码
3. **执行权限**：确保脚本有执行权限
4. **输出规范**：脚本执行结果会显示在日志中

示例 BAT 脚本：
```bat
@echo off
echo 脚本开始执行
echo 当前时间：%date% %time%
ping -n 3 127.0.0.1 > nul
echo 脚本执行完成
pause
```

## 故障排除

### 常见问题

#### 1. 程序无法启动
**问题**：提示 "未安装 PyQt5"
**解决**：重新安装依赖包
```bash
pip install PyQt5 requests flask
```

#### 2. 本地连接失败
**问题**：无法访问 `http://127.0.0.1:8888/webhook`
**解决**：
1. 检查程序是否正常运行
2. 检查端口 8888 是否被其他程序占用
3. 重启程序

#### 3. 外部连接失败
**问题**：无法访问 `http://117.72.35.239:6000/webhook`
**解决**：
1. 检查 FRP 是否正常启动
2. 检查 FRP 服务端配置是否正确
3. 运行测试脚本诊断问题
```bash
python test_connection.py
```

#### 4. BAT 脚本不执行
**问题**：webhook 请求成功但脚本未执行
**解决**：
1. 检查 BAT 文件是否存在
2. 检查文件路径是否正确
3. 检查文件编码格式
4. 检查脚本是否有语法错误

## 高级配置

### 修改监听端口
如果需要修改 webhook 服务的监听端口：

1. 编辑 `config.json` 文件
2. 修改 `listen_port` 字段的值
3. 重启应用程序

### 添加新的 FRP 配置
1. 在主界面点击 "FRP 设置"
2. 点击 "添加配置"
3. 填写新的配置参数
4. 保存并启动

### 自动启动
如果需要开机自动启动：

1. 创建快捷方式到启动文件夹
